物联网固件安全

发表于2024-10-23|更新于2024-10-23|固件安全
|浏览量:

一、物联网硬件设备组成

1、印制电路板（PCB）

​ 是电子元器件的支撑体，用来实现各个元器件之间的电器连接。

2、存储芯片

​ 只读存储器（ROM），只能读出事先存储的数据。

​ PROM：可编程只读存储器

​ EPROM：可擦除可编程只读存储器

​ EEPROM：电擦除可编程只读存储器

​ FLASHROM：快闪存储器

3、电阻电容

二、物联网通信接口

1、通用异步收发器（UART）

​ 发送端UART将来自CPU等控制设备的并行数据转换为串行形式，并将其发送给接收端UART，接收端UART将串行数据换回并行数据，并发送给接受设备。

​ 主要用于主机和辅助设备之间的通信。


​ 为了方便调试，大部分iot设备会将UART引脚引到PCB中。


​ 四个引脚：VCC、GND、Rx、Tx。UART接口的通信可以理解为：通过对芯片具有不同功能的引脚输入不同的高低电平，完成对主控设备的控制与调试。

2、联合测试工作组（JTAG）

​ 主要用于芯片内部的测试以及对系统进行仿真、调试。JTAG是一种嵌入式调试技术，在芯片内部封装了专门的测试电路TAP（测试访问端口），通过专门的JTAG测试工具对内部节点进行测试。

​ 标准JTAG接口定义了一些信号管脚：TMS、TCK、TDI、TDO、TRST、STCK、DBGRQ。提供多种接口标准，常见有10针、14、16、20针等。

3、通过万用表识别UART引脚

​ 识别GND。


​ 识别VCC。


识别RX、Tx引脚。

4、用自动化工具JTAGulator进行识别

​ 是一款可帮助识别目标设备引脚的开源硬件。

​ 识别Tx和Rx引脚时，先识别GND引脚，然后连接其他引脚，再通过USB连接到计算机。

​ 用SecureCRT工具与JTAGulator交互，进入交互模式。

5、识别UART波特率

​ 波特率：UART设备在通信时，数据的传输速率。

​ （1）CH340接口+SecureCRT

​ 使用CH340连接到计算机的USB接口，在SecureCRT软件中多次切换波特率，当交互界面输出正常字符时，此时选的就是当前UART设备的波特率。

​ （2）使用JTAGulator进行识别

​ 在SecureCRT页面设置波特率为115200，即可进入交互模式。

​ 在交互界面输入u命令，可在输出对应引脚的同时完成波特率识别。

6、连接与调试通信串口

​ CH340接口转换芯片对设备的串口进行调试。

​ 通过输出数据可以看到UART设备完整的启动过程。

三、固件分析

​ 固件（Firmware）是一类特定计算机软件，可为特定硬件提供底层控制。

1、常见的文件系统

  • SquashFS：供linux内核使用的只读压缩文件系统。（适用长期开机且对稳定性要求高的系统）

  • JFFS2：日志闪存文件系统（适用两种闪存介质的嵌入式系统）

  • YAFFS2：一个专门为NAND闪存设计的文件系统。

  • UBIFS：无序区块镜像文件系统，主要用在闪存。

  • CramFS：专门针对闪存，只读、压缩文件系统····

2、获取固件

官网获取固件

​ 从设备厂商官网下载固件。

流量拦截获取固件

​ 在设备升级时，对设备或者控制设备的APP端进行流量代理转发，实施中间人攻击。

​ 用到工具：Ettercap、Wireshark、tcpdump。这些工具集成在Kali Linux中。

  1. kail启动ip转发功能
  2. 启动ettercap图形界面，选择当前网卡
  3. 图形界面找到scan for hosts选项，扫描内网设备
  4. 在扫描后的内网设备的IP和MAC地址列表中，选择当前网络网段地址Add to Target 1，选择要欺骗的设备的IP地址Add to Target 2
  5. 在图形界面菜单栏右侧点击地球图标，选择ARP poisoning，进行ARP欺骗攻击相关配置。然后在弹出的窗口中选中Sniff remote connections复选框，点击ok完成
  6. 在APP端进行升级，然后在kali linux系统上执行tcpdump命令抓取APP端流量
  7. 抓到流量载入wireshark进行流量分析
  8. 根据更新的版本号，进行udp过滤筛选，获取第一次通信数据包追踪流
  9. 在弹出的界面中查找形如地址字样的字符即可

==ARP欺骗的配置流程回顾==

  • Target 1： 设置为网关的IP地址。

  • Target 2： 设置为受害设备的IP地址。

  • ARP Poisoning选项： 勾选”Sniff remote connections”，表示要嗅探目标设备的所有连接。

  • 通过这个设置，Ettercap会发送伪造的ARP包给网关和受害者，迫使双方把攻击者误认为是彼此的通信对象。这一过程无需用户交互就完成了，因为ARP是无状态的协议，不会验证收到的ARP响应包的真实性。

编程器从闪存读取固件

物理提取，拿夹子夹住闪存提取固件。保存为bin文件。

串口调试获取固件

  1. 将FT232芯片与要进行串口调试的嵌入式设备通信接口连接，同时将FT232连接到电脑
  2. 打开minicom或者securecrt工具，设置好串口调试接口和波特率，然后开始监听
  3. minicom启动屏幕捕获功能
  4. 设备接电启动，设备启动过程中按下ctrl+c进入u-boot模式
  5. 在U-boot模式下输入？可以显示所支持的功能和命令，mem、flash命令
  6. 执行flash -layout命令查看闪存布局情况，重点读取固件区域：执行flash read命令将固件读取到内存中的指定空闲位置
  7. 执行mem -dump命令查看Firmware是否被存入指定位置，之后循环读取实际大小即可，最后将得到16进制数据保存为bin格式为文件

3、提取固件中的文件系统

（1）系统自带命令提取文件系统

​ 固件中包含Boot Loader、内核、文件系统以及其他内容，其中文件系统包含了需要研究的大部分内容，包括web应用、协议、核心控制程序等，文件系统时着重研究对象。

  1. hexdump查看.bin文件的十六进制编码，然后使用grep命令在结果中检索hsqs字符串
  2. 使用 dd命令提取固件，dd if=输入文件名 bs=读入/输出大小 skip=从文件头开始跳过块数 of=输出文件名
  3. 执行file命令查看解压文件系统是否正确
  4. 执行unsquashfs命令，提取出整个文件系统

（2）工具提取（binwalk）

​ 1、安装binwalk工具

​ 2、安装binwalk需要的所有依赖文件和二进制文件

​ 3、查看固件详细信息

​ 4、自动提取文件系统，-M表示递归分解扫描出来的文件，-e表示按照定义的配置文件中的提取方法从固件中提取探测到的文件系统

​ 5、binwalk提取后的文件系统以_固件名.extracted命名，打开iot.bin.extracted文件夹，squashfs-root里面存放的就是获取的文件系统

4、分析文件系统

（1）使用firmwalker分析文件系统

​ firmwalker是一个简单的bash脚本，用于搜索和分析固件文件系统。利用linux系统自带的系统命令，查找固件的文件系统，然后罗列出包含敏感字符的文件（敏感字符可定制）。

​ 执行以上命令后，firmwalker会将分析完的结果输出到当前工作日目录下.txt，会将包含passwd、pwd等敏感字符全部列举出来

（2）使用trommel工具分析文件系统

​ trommel是一款使用python编写的开源工具。

​ trommel检索到的内容比firmwalker更多，还将php文件中shell、system等也列举了出来

（3）使用emba自动分析固件

​ emba是一款开源的固件扫描软件。

​ emba会把二进制文件中危险函数列举出来，还可以用来检索CVE漏洞库，从中搜索相应组件或应用在过去出现过的CVE漏洞的编号。

（4）使用FACT自动分析固件

​ FACT是一款拥有Web界面的自动化固件测试平台。

​ 可以显示固件包含组件中曾经有哪些历史漏洞。

5、固件模拟

（1）QEMU固件模拟

​ 两种运行模式：用户模式、系统模式

​ 1、使用QEMU用户模式来模拟

  • 执行 cp命令把针对小端MIPS架构的QEMU二进制运行文件qemu-mipsel-static复制到squashfs-root文件夹下

  • 使用chroot命令执行QEMU虚拟机，并以模拟方式运行二进制文件

    2、使用QEMU系统模式进行模拟

  • 在系统模式下，需要为QEMU指定==内核镜像、IDE硬盘镜像、内核参数==。

  • 去Debian官网下载相应版本，以及对应要求下载内容。

  • 配置虚拟网卡，让QEMU虚拟机能够和主机进行网络通信。

  • 执行qemu-system-mipsel命令启动QEMU虚拟镜像并设置与主机通信

  • 使用tar -zcvf squashfs-root.tar.gz sqashfs-root命令打包之前解压的squashfs-root文件夹，并在当前文件夹下启动HTTP服务供QEMU虚拟机下载当前固件使用

  • 开启HTTP服务后，在QEMU虚拟机中执行wget命令下载并解压文件

  • 挂载/dev、/proc目录，执行chroot squashfs-root sh命令，切换squashfs-root为根

（2）Firmware Analysis Tooklkit模拟

​ FAT（固件分析工具集）是一个自动化脚本，使用firmadyne工具来模拟固件，firmadyne底层基于QEMU，还集成了其他一些脚本，包含binwalk等工具。

  • 从GitHub上安装，./setup.shFAT会自动下载所需要的软件库
  • 简单配置，打开fat.config将sudo_password修改为当前用户密码
  • 在FAT目录下运行./fat.py脚本，后接固件的存放路径，==即可进行模拟==
  • 在自动配置好FAT程序初始化、固件文件系统提取、镜像创建、网络配置等一序列操作后，FAT会分配一个IP，浏览器输入IP即可进入路由器界面

（3）FirmAE模拟

​ FirmAE是在firmadyne基础上开发的一种执行仿真和漏洞分析的全自动框架。引入了仲裁仿真方式，从固件启动、网络、NVRAM、内核以及其他5个方面，总结了导致固件仿真失败的原因并给出具备通用性的方法。

  • 安装依赖文件./downlaoad.sh，安装脚本./install.sh，初始化操作./init.sh
  • 下载固件到FirmAE目录下的firmwares目录内
  • 执行run.sh脚本对固件进行模拟
  • FirmAE会先自动分配一个虚拟网卡，并配置相关IP，然后开始模拟固件
  • 输入分配的IP地址，即可访问对应路由器的管理界面

6、固件解密

（1）解密方法1

​ 设备在出厂时未加密v1.0，解密程序与未加密版本的新版本固件为v1.1，此后发布的固件为加密固件v1.2。我们可以从固件v1.1中获取解密程序，用它来解密v1.2版本固件。

（2）解密方法2

​ 设备固件在原始版本中进行了加密，厂商决定更改加密方案并发布一个未加密的新固件v1.2，其中包含了新版本解密程序。

​ 在更新固件版本前，需要先看新固件版本的发布通告，固件升级到最新版前，需要先升级到固件一个中间版本，中间版本是未加密的固件版本，通过中间版本固件进行升级可以获得解密程序。

（3）解密方法3

​ 原始版本进行了加密，厂商决定更改加密方案并发布一个带新版解密程序的中间版迭代加密固件，这时候很难获得解密程序。

​ 对加密后的固件解密很困难，这种情况一种思路是，购买设备并使用JTAG、UART调试等方式，从设备硬件中提取固件中的文件系统，对固件进行更深层次的分析，尝试对加密固件进行破解。

（4）对加密固件解密

​ binwalk -E命令查看该固件的熵值。熵值计算是确认给定的字节序列是否压缩或加密的有效手段。熵值越大越有可能是加密或者压缩过的。

  • 查看官网固件升级文档中的信息，去下载中间版本固件
  • 用grep命令在中间版本固件中搜索update、firmware、upgrade、download等关键字符
  • 通过筛查关键字符，找到关键下载文件，从该文件中找到固件解密的代码信息
  • 按照提示进行操作，查找相应文件和参数，在QEMU下模拟
  • 模拟后即可对加密固件解密，提取文件系统
